001 / Введение

Кто это? Определение и границы ответственности

Frontend-разработчик — специалист, который реализует пользовательский интерфейс веб-приложения. Он берёт дизайн-макет и превращает его в код, работающий в браузере. В зону ответственности входит: вёрстка, логика интерфейса, работа с данными от сервера, оптимизация загрузки, обеспечение доступности (a11y) и кросс-браузерной совместимости.

В отличие от верстальщика (который только раскладывает блоки по сетке), фронтенд-разработчик пишет полноценную клиентскую логику: управляет состоянием приложения, обрабатывает пользовательские сценарии, работает с API, настраивает сборку и деплой. В малых командах фронтендер часто берёт на себя ещё и задачи UI/UX-инженера.

Типичные заблуждения: фронтенд — это не «лёгкий вход в IT» и не «только вёрстка». Современный фронтенд включает древовидную архитектуру компонентов, нюансы рендеринга (CSR / SSR / SSG / ISR), бандлинг, динамический импорт, теневой DOM, протоколы реального времени и интеграцию с AI-сервисами на стороне клиента.

003 / Обязанности

Что реально делает? Не «пишет код», а решает проблемы

01

Строит интерфейсы

Собирает приложение из компонентов. Управляет состоянием (через Redux, Zustand, Pinia или signals во Vue). Настраивает роутинг (React Router, TanStack Router). Следит, чтобы интерфейс не перерисовывался без необходимости — мемоизация, виртуализация списков (react-window, TanStack Virtual), оптимизация ре-рендеров.
02

Работает с данными

Ходит в API (REST / GraphQL), кеширует ответы (React Query / SWR / Apollo), обновляет данные в реальном времени через WebSocket или Server-Sent Events. Обрабатывает загрузку, ошибки, ретраи. Делает optimistic updates, чтобы интерфейс не ждал сервер.
03

Оптимизирует

Следит за Core Web Vitals: LCP (самая большая картинка/текст), CLS (сдвиги макета), INP (задержка на клик). Дробит бандл через code splitting, грузит тяжёлые компоненты лениво (lazy loading), выносит CSS в отдельные чанки. Сжимает изображения через WebP/AVIF. Настраивает кеширование через Service Worker.
04

Обеспечивает качество

Пишет тесты: модульные (Vitest, Jest), компонентные (Testing Library, Vue Test Utils), E2E (Playwright, Cypress). Внедряет линтеры (ESLint, stylelint) и форматтеры (Prettier). Проводит code review. Проверяет доступность: семантическая разметка, aria-атрибуты, навигация с клавиатуры, контрастность (WCAG 2.1 AA).

004 / Стек

Чем работает? Не просто список технологий, а что зачем

Языки

  • JavaScript (ES2024+)
  • TypeScript (де-факто стандарт)
  • HTML5 (семантика, a11y)
  • CSS3 (Grid, Container Queries)
TypeScript — обязателен в 80% вакансий. Не знать TS = отсев на этапе резюме.

Фреймворки

  • React 19 + Next.js / Remix
  • Vue 3 + Nuxt
  • Angular 17+ (standalone API)
  • Svelte 5 + SvelteKit
  • Solid.js / Qwik
React лидирует по числу вакансий. Vue — популярен в Европе и Азии. Svelte и Solid — растущий тренд.

Инфраструктура

  • Vite — сборка (заменил CRA)
  • Turbopack / Webpack — legacy
  • Tailwind CSS — утилитарный CSS
  • ESLint + Prettier
  • Husky + lint-staged
Vite — стандарт для новых проектов. Tailwind используется в 60% опросов (2024). CRA официально deprecated.

005 / Карьера

Путь в профессии От джуна до лида — что меняется

  • Junior Может сверстать макет по Figma, пишет простые компоненты, знает основы JS и один фреймворк. Нужен код-ревью. Обычно не отвечает за архитектуру. Задачи: багфикс, мелкие фичи, вёрстка страниц.
  • Middle Уверенно работает с фреймворком, настраивает сборку, пишет тесты, проводит ревью. Понимает, когда нужен SSR, когда SPA, а когда вообще хватит статики. Рефакторит чужой код без страха.
  • Senior Проектирует архитектуру фронтенда, выбирает стек, оптимизирует производительность. Участвует в оценках и планировании. Знает бэкенд на уровне «что ему нужно от API». Умеет объяснять бизнесу, почему техдолг надо отдавать.
  • Lead / Head Управляет командой фронтенда, нанимает, онбордит, выстраивает процессы. Пишет код редко — в основном code review и прототипы. Отвечает за результат, а не за строчки.

006 / Тренды

Что прямо сейчас Куда движется индустрия

  • Server Components React Server Components (RSC) меняют подход: часть компонентов рендерится на сервере, никогда не отправляя JS на клиент. Меньше бандла, быстрее загрузка.
  • AI в интерфейсе Чат-интерфейсы, AI-автодополнение, генерация UI по промпту. Stream через SSE — стандартный паттерн. AI-агенты и веб-интерфейсы для LLM.
  • WebGPU API для вычислений на GPU в браузере. Позволяет делать тяжёлую 3D-графику, симуляции, обработку видео и ML-инференс на клиенте без плагинов.
  • Меньше JS Тренд на уменьшение JavaScript-бандла: island architecture (Astro), partial hydration (Qwik), компилируемые фреймворки (Svelte, Solid).

Источники: State of JS 2024, HTTP Archive, Web Almanac, отчёты JetBrains Developer Ecosystem.

007 / Мифы

Чего не надо говорить Популярные заблуждения о фронтенде

❌ «Это же просто вёрстка»

Фронтенд включает управление состоянием, оптимизацию производительности, тестирование, CI/CD. Вёрстка — 20–30% работы.

❌ «Можно без математики»

Анимации, гайки, алгоритмы поиска/фильтрации, сложность O(n), работа с канвасом — математика нужна регулярно.

❌ «Фреймворки — одно и то же»

React — библиотека без мнения об архитектуре. Vue — фреймворк с магией. Svelte — компилятор. Solid — без виртуального DOM. Разный подход к реактивности.

❌ «Достаточно знать React»

React — только инструмент. Без понимания DOM, HTTP, accessibility, рендеринга, бандлинга — вы просто пользователь API.
